# 一、顶部 tab 栏（标签条）

## 共通规格

| 项 | 值 |
|---|---|
| 单个 tab 高 | **44** |
| Home tab（选中态固定第一位） | 宽 **200** |
| 普通站点 tab（默认宽度） | 宽 **200** |
| 长标题 tab（如 "Stream ESPN+ Live…"） | 宽 **282** |
| 多标签收窄态最小宽 | **120** |
| tab 间分隔线 | 1px × 16, `#292929`, 圆角 1，位于 tab 右内侧 |
| tab 文字 | Arial Regular 14 / 20  `#aeb1b6` |
| tab 内左 padding（icon 左距） | 16 |
| tab 内 icon 尺寸 | 16 × 16 |
| icon 与文字间距 | 8 |
| 选中态背景 | 使用 `选中bg`（imgBg1）作为底图（图片，非纯色） |
| 未选中背景 | 使用 `未选中bg`（imgBg3 / img1 / imgBg4）作为底图 |
| 选中态附加底部小三角指示 | `选中bg备份 2` 12 × 10 三角，位于 tab 下沿中部 |

> 备注：tab 文字色统一 `#aeb1b6`，**选中态没有改变文字色**（按设计稿落地）。选中视觉差异主要来自背景图（深色块 vs 浅色块）和下沿小三角。

## 4 种状态差异

### 1. 默认状态（`0:4481`）

```
[Home (选中)]  [ Stream ESPN+ Live…              ]  [ Netflix ]  [ Netflix ]
   200×44          282×44                              200×44       200×44
```

- 长 tab 内：左侧 16×16 站点 logo + 文字（截断 "Stream ESPN+ Live…"），**无右侧操作 icon**
- Netflix tab：左 16×16 logo + 文字 "Netflix" + 右侧 1px 分隔条

### 2. Hover 状态（`0:4618`）

与默认完全一致，**仅长 tab 右侧多了一个 `icon_close` (16×16) 关闭 X**，距 tab 右边 28。

- close X 节点：`0:4645`
- 仅在鼠标 hover 该 tab 时显示，离开消失

### 3. 视频录制中（`0:4669`）

与默认完全一致，**仅长 tab 右侧多了一个录制中指示 icon (`icon_right`, 16×16)**，距 tab 右边 28（同 close X 位置）。

- icon 节点：`0:4696`
- 图标内容：实心小圆 + 外圈刻度（"录制中"指示符）
- 用于表明"这个 tab 当前正在被 RecordFab 录制"，与 close X **互斥**（录制中时不显示 close X）

### 4. 多个标签（`0:4529`）

当标签数量过多容器塞不下时：

- 所有 tab **统一收窄到宽 120**（包括 Home 选中态）
- tab 内：icon 16×16 + 文字（如 "Netflix"），居中或近中显示
- 容器**右端追加一对左右翻页箭头**：
  - 左箭头 `icon_left` (`0:4532`) 16×16，路径 6×12 三角，bg 透明
  - 右箭头 `icon_left备份` (`0:4535`) 16×16，垂直镜像左箭头得到（设计稿用 `-scale-y-100 rotate-180` 表达）
  - 两箭头间距 32，距容器右边视容器宽而定

> 触发条件（推测）：当 tab 数 × 默认宽 > 容器可视宽时，触发收窄；当所有 tab 在最小宽 120 下仍塞不下时，启用翻页箭头分页。具体触发阈值需与开发约定。

## 4 种状态差异

### 状态 1：未检测到视频（`0:4723`）

| 元素 | 值 |
|---|---|
| 左 icon | `icon_loading`（旋转环 / 加载动画） |
| 文案 | `Detecting Video...` |
| Record 按钮 | `#6d87ff` 浅蓝 **disabled** |

> 含义：刚进入站点 / 切换 tab，正在自动嗅探视频流。Record 不可点。

### 状态 2：检测到视频 - 可录制（`0:4735`）

| 元素 | 值 |
|---|---|
| 左 icon | `icon/right`（实心圆 + 对勾） |
| 文案 | `Video Detected` |
| Record 按钮 | `#4162fb` 主蓝 **可点** |

> 含义：嗅探到可录制的视频流，可以开始录制。Record 高亮主蓝。

### 状态 3：检测到视频 - 无法录制（`0:4747`）

| 元素 | 值 |
|---|---|
| 左 icon | `icon/right`（实心圆 + 对勾）— 与状态 2 一致 |
| 文案 | `Video Detected` — 与状态 2 一致 |
| Record 按钮 | `#6d87ff` 浅蓝 **disabled** |
| 附加：浮窗 tooltip | **264 × 28**，bg 深色，文字 `Refresh the page or it is not supported` Arial 14/20 `#eee` 居中 |
| 浮窗位置 | 位于状态条**上方**，右对齐到 Record 按钮附近（设计稿示意 left 864, top 582，相对该状态条上沿往上 32px） |

> 含义：检测到视频元素但当前页面状态/DRM/编解码等原因无法启动录制。**视觉与状态 2 几乎完全一致**，差异仅在按钮变浅 + 出现 tooltip。落地时建议 tooltip 在 hover Record 按钮时浮出，不要常驻。

### 状态 4：录制中（`0:4759`）

| 元素 | 值 |
|---|---|
| 左 icon | `icon_right`（"录制中"指示符，实心圆 + 外圈刻度，与顶部 tab 录制中 icon 是**同一资源**） |
| 文案 | `Video Recording` |
| Record 按钮 | `#6d87ff` 浅蓝 **disabled** |

> 含义：当前正在录制，Record 按钮锁定。结束录制由"录制中蒙层"（`0:6470`）上的 Stop 控制。

## 不在本节点的内容

- 录制工作区的浏览器视口、地址栏、视频播放区（在 `0:6354`）
- 录制中视频画面上的蒙层（在 `0:6470`）
- 点 Record 后弹出的 Trial 提示（`0:7754`）和 Record Setting（`0:7911`）
- 多标签翻页箭头点击后的 tab 滚动 / 分页逻辑

## 落地建议

- **按钮颜色二态化**：`#4162fb`（可点） vs `#6d87ff`（disabled）。除"检测到视频-可录制"外，其他三个状态 Record 都是 disabled
- **顶部 tab 录制中 icon 与底部录制中 icon 是同一个资源**，下载时只取一份
- **tooltip "Refresh the page or it is not supported"** 建议做成 hover Record 时浮出的 popover，而不是常驻；常驻会与正常的"可录制态"视觉相似导致用户误读
- **多标签收窄阈值**：默认 200 → 收窄到 120 是设计稿给出的两个固定档位，中间不需要按比例平滑收窄；超出 120 仍塞不下时启用翻页箭头
- **Hover close X 与录制中 icon 是互斥关系**：录制中的 tab 不应允许 hover 出 close X（防止用户误关）
